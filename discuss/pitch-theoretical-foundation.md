# Part I: 产品侧 — Career DNA 画像与上下文获取

## 核心理念

传统招聘用简历和 JD 做文本匹配，测的是"你做过什么"。
TalentDrop 用行为问卷和深度画像做人格匹配，测的是"你是什么样的人"。

这个思路不是我们发明的——Date Drop 在斯坦福用同样的逻辑做约会匹配，实际约会转化率是 Tinder 的 10 倍。我们把它迁移到招聘场景。

---

## 1. Big Five / OCEAN 模型 — Career DNA 维度设计的理论起点

心理学界花了几十年争论，最终在 1990 年代达成共识：**人的性格可以用 5 个独立维度描述**。

| 维度 | 高分 | 低分 | 与职场的关系 |
|------|------|------|-------------|
| **O**penness 开放性 | 好奇、爱尝鲜、抽象思考 | 务实、偏好熟悉事物 | 创新型岗位的正向预测因子 |
| **C**onscientiousness 尽责性 | 自律、有条理、靠谱 | 随性、灵活、易拖延 | **几乎所有岗位绩效的最强单一预测因子** |
| **E**xtraversion 外向性 | 社交充电、主动表达 | 独处充电、安静倾听 | 销售/管理岗位的正向预测因子 |
| **A**greeableness 宜人性 | 温和、合作、信任 | 直接、竞争、怀疑 | 团队合作的正向预测因子 |
| **N**euroticism 神经质 | 易焦虑、情绪波动大 | 情绪稳定、抗压 | 工作满意度的负向预测因子 |

**我们没有直接用 Big Five**，而是将其"场景化"为 8 个职场维度。例如 Big Five 的 E（外向性）太宽泛，我们窄化为 Collab（协作模式）；Big Five 中跨岗位最强的绩效预测因子 C（Conscientiousness）直接映射为 Execution（执行风格）。

---

## 2. Career DNA 8 维模型 — 维度设计与理论锚点

| Career DNA 维度 | 光谱 | 理论锚点 | 关系说明 |
|----------------|------|---------|---------|
| Pace 工作节奏 | 快速迭代 ↔ 深度打磨 | Big Five — Extraversion 活力子维度 | 职场特有，通过效度验证确认预测力 |
| Collab 协作模式 | 独立驱动 ↔ 团队协同 | Big Five — Extraversion + Agreeableness | 更窄更聚焦于工作协作场景 |
| Decision 决策风格 | 数据驱动 ↔ 直觉判断 | Cognitive Style Theory | 与分析-直觉认知风格光谱相关 |
| Expression 表达风格 | 直接开放 ↔ 策略含蓄 | **TKI 果断性轴 + Big Five Agreeableness** | 合并原 Comms+Conflict，沟通与冲突在数据层不可分 |
| Unc 不确定性容忍 | 拥抱模糊 ↔ 追求明确 | Big Five — **Openness** | 最接近已有维度，高度重叠 |
| Growth 成长路径 | 专精深耕 ↔ 广泛涉猎 | T-shaped vs I-shaped | 有行业讨论但无标准量表 |
| Motiv 驱动力来源 | 使命价值 ↔ 回报激励 | **Self-Determination Theory** | 学术基础最扎实的一个维度 |
| Execution 执行风格 | 计划严谨 ↔ 灵活应变 | **Big Five — Conscientiousness** | 跨岗位最强绩效预测因子 (Barrick & Mount 1991, r≈0.22-0.27) |

**关键设计原则：** 所有维度都是光谱（两端都合理），不存在"正确答案"。这与 Pymetrics 的 91 个连续谱特征设计一致。

## 3. Self-Determination Theory (SDT) — Motiv 维度的学术后盾

Deci & Ryan (1985) 提出，人的内在动机来自三个基本心理需求：

| 需求 | 含义 | 职场表现 |
|------|------|---------|
| **Autonomy** 自主感 | "我能决定自己做什么" | 追求自主权、抗拒微管理 |
| **Competence** 胜任感 | "我能搞定这件事" | 追求挑战、技能成长 |
| **Relatedness** 归属感 | "我被需要、被认可" | 重视团队关系、组织认同 |

Career DNA 的 Motiv 维度（使命价值 ↔ 回报激励）对应的是：
- **使命价值端** → 更多受 Autonomy + Relatedness 驱动（"做这件事因为它有意义"）
- **回报激励端** → 更多受外在动机驱动（"做这件事因为回报好"）

---

## 4. Thomas-Kilmann Conflict Mode (TKI) — Expression 维度的学术后盾

组织行为学中测量冲突处理风格的**标准工具**，用两个维度定义 5 种风格：

```
        果断性 (Assertiveness)
            ↑
    竞争     │     协作
  Competing │   Collaborating
            │
  ──────────┼──────────→ 合作性 (Cooperativeness)
            │
    回避     │     迁就
   Avoiding │   Accommodating
            │
         妥协 Compromising（中间）
```

我们将 TKI 的**果断性轴**与 Big Five 的 Agreeableness 结合，定义 Expression（表达风格）维度：直接开放 ↔ 策略含蓄。
- **合并理由：** 原 Comms（沟通偏好）和 Conflict（冲突处理）在题目数据层面不可分——Q04/Q08 每个选项中两者编码方向完全同向
- **简化的代价：** TKI 合作性轴被简化，"协作"和"妥协"风格在单光谱中自然落在中间位置
- **简化的好处：** 消除维度冗余，腾出位置给更有预测力的 Execution 维度

## 4.5. Conscientiousness — Execution 维度的学术后盾

Big Five 中的 **C（Conscientiousness，尽责性）** 是学术界跨岗位、跨行业最强的绩效预测因子：

- Barrick & Mount (1991) 元分析：across all job types, r ≈ 0.22-0.27
- 高尽责性 → 主动规划、承诺必达、结构化工作流、注重文档
- 低尽责性 → 随机应变、弹性安排、快速响应变化、轻流程

**两端都合理**——高度计划严谨在运维/金融/安全岗至关重要，灵活应变在早期创业/探索研究不可或缺。这是 Career DNA 光谱设计的核心理念。

## 5. 信度与效度 — 问卷质量的两把尺子

### 信度 (Reliability) = 稳定性

> 同一个人，两周后重新做问卷，结果一样吗？

```
第一次测：  Pace=72, Collab=45, Decision=88
两周后重测：Pace=70, Collab=47, Decision=85

重测相关系数 r = 0.92 → 信度高 ✓
如果 r < 0.7 → 题目测的可能是"心情"而不是"性格" ✗
```

**类比：** 体重秤今天量 70kg，明天 73kg，后天 68kg——信度太低，不可信。

### 效度 (Validity) = 准确性

> 问卷测出来的东西，真的能预测你想预测的结果吗？

```
想预测：90 天留存率
测出来：Career DNA 8 维评分

效度 = correlation(匹配分, 90天后真实满意度)
> 0.3 → 效度可接受
< 0.1 → 问卷可能在测一个和留存无关的东西
```

**类比：** 发明了一把"体重秤"，每次量出来都很稳定（信度高），但实际测的是身高——信度高效度为零。

### TalentDrop 的策略

Demo 阶段不需要证明信效度（缺乏数据）。叙事重点：**"维度模型基于成熟理论设计，将通过 90 天真实结果数据做信效度验证，持续淘汰低效维度。"**

---

## 6. 维度正交性 — 8 维是否在测 8 件不同的事

两个维度"正交" = 它们之间**不相关**，测的是完全不同的东西。

**已完成的优化（v2）：**

原设计中 Comms（沟通偏好）和 Conflict（冲突处理）共享完全相同的主测题（Q04、Q08），数据层面不可区分。已合并为 Expression（表达风格），腾出维度位给 Execution（执行风格，Conscientiousness）。

**剩余监控项：**

```
Decision（数据驱动 ↔ 直觉判断） ←→ Unc（拥抱模糊 ↔ 追求明确）
→ 潜在相关 r≈0.3，但各自有 3+ 道纯净题，分离能力足够
→ 需要积累 1000+ 用户数据后跑因子分析 (Factor Analysis) 验证
```

## 7. Person-Organization Fit (P-O Fit) — 匹配算法的理论根基

**核心命题：** 人和组织的"契合度"能预测工作满意度、绩效和离职率。

### 两种契合

| 类型 | 含义 | 示例 |
|------|------|------|
| **补充型 Supplementary** | "我和团队很像" | 我重视创新，团队也重视创新 |
| **互补型 Complementary** | "我补上团队缺的" | 团队全是技术人缺沟通，我沟通强 |

### 为什么不能只算"相似度"

```
简单相似度：|候选人分 - 企业分| → 越小越好
问题：只能捕捉补充型契合，忽略互补型

Edwards 多项式回归（学术标准方法）：
Z = b₀ + b₁P + b₂E + b₃PE + b₄P² + b₅E²
→ 交互项 PE 能同时捕捉"像 = 好"和"互补 = 好"两种模式
```

冷启动阶段用简化的距离度量，数据充足后升级为 Edwards 模型。

## 8. 几何平均 — 双向匹配的数学保障

招聘是双向的：候选人再喜欢公司，公司不要他也没用。

```
候选人对岗位满意度：90%
岗位对候选人满意度：10%

算术平均：(90 + 10) / 2 = 50%  ← 一方不满意，仍得 50%？
几何平均：√(90 × 10) = 30%     ← 更真实反映匹配质量差

极端情况：
算术平均：(100 + 0) / 2 = 50%  ← 一方完全不想要，还给 50%？
几何平均：√(100 × 0) = 0%      ← 任何一方为零，匹配就是零 ✓
```

**核心直觉：** 几何平均天然惩罚"单相思"，确保匹配必须是双向的。OkCupid 公开验证了这一方法的有效性。

## 9. Gale-Shapley 稳定匹配 — Weekly Drop 的配对优化算法

2012 年诺贝尔经济学奖（Alvin Roth & Lloyd Shapley），原论文 1962 年。

### 用故事解释

```
3 个求职者 (A, B, C) 和 3 个岗位 (X, Y, Z)，每人都有偏好排序：

  A: X > Y > Z    X: B > A > C
  B: Y > X > Z    Y: A > B > C
  C: X > Y > Z    Z: A > B > C

第一轮：每个求职者向最想去的岗位"提议"
  A → X, B → Y, C → X
  X 收到 A 和 C → 更喜欢 A → 暂时接受 A，拒绝 C
  Y 收到 B → 暂时接受 B

第二轮：被拒的 C 向第二志愿提议
  C → Y → Y 对比 B 和 C → 更喜欢 B → 拒绝 C

第三轮：C → Z → Z 接受

最终：A-X, B-Y, C-Z ← 稳定匹配
```

**"稳定"的含义：** 不存在任何一对双方都觉得"我更想要对方而不是现在的匹配"。如果存在这样的一对，匹配就不稳定——他们会"私奔"。

**在 TalentDrop 中的应用：** 候选人不可能手动排序所有岗位，因此用 L2-L4 的综合评分**代替**显式偏好排序，再运行 Gale-Shapley 算出每周最优配对。

**行业验证：** Hinge 的 "Most Compatible" 基于 Gale-Shapley，匹配成功率是其他推荐的 **8 倍**。Date Drop 同样以此为算法基础，实际约会转化率是 Tinder 的 **10 倍**。

## 10. LightRAG — 匹配智能体的"认知大脑"

LightRAG（港大 HKUDS, EMNLP 2025）是一个将知识图谱与 RAG 深度融合的框架。它在 TalentDrop 中**不做匹配算法本身**，而是做匹配算法的**上下文理解层**。

### 它解决什么问题

Career DNA 问卷产出结构化评分（8 个维度各一个数字），但 TalentDrop 还有大量**非结构化文本**无法被简单量化：

| 非结构化数据 | 蕴含的信号 |
|-------------|-----------|
| 候选人 Dream Role 描述 | 真实职业期望、表达风格 |
| AI 语音对话转录 | 沟通风格、价值观的自然流露 |
| 企业员工匿名评价 | 真实文化而非 HR 话术 |
| Glassdoor / Blind 评价 | 第三方文化验证 |
| 推荐人的推荐理由 | 社交信任信号 |

LightRAG 把这些文本转化为**实体 + 关系的知识图谱**，使匹配算法能"理解"数字之外的信息。

### 双层检索

```
低层检索（精确）："候选人 A 具体擅长什么？"
→ Person_A → [masters] → {React, TypeScript, GraphQL}
→ 扩展：React → [related_to] → {Next.js, Redux}

高层检索（宏观）："什么样的人在快速迭代文化中表现最好？"
→ Culture_快速迭代 ← [embodies] ← {Team_X, Team_Y}
→ 这些团队中高绩效者的共同特征是什么？
→ "拥抱模糊 + 直接坦率"维度高分者留存率高 40%
```

### 增量更新

每次有新数据（新用户问卷、新的 90 天反馈、新的员工评价）直接并入已有图谱，**无需重建**。这是 LightRAG 相比 GraphRAG 的核心优势。

## 11. 进化机制 — 智能体如何越来越聪明

### 三阶段进化路径

| 阶段 | 数据量 | 匹配策略 | LightRAG 角色 |
|------|-------|---------|--------------|
| **冷启动** (0-1K 用户) | 问卷数据为主 | 基于规则+嵌入的内容匹配 | 处理开放文本，生成匹配报告 |
| **数据积累** (1K-10K) | 有结果反馈 | 完整 5 层算法 + Gale-Shapley | 增量更新图谱，发现文化模式 |
| **智能进化** (10K+) | 结果数据充足 | 协同过滤上线，权重自动调优 | 多跳推理发现隐式成功因素 |

### 反馈驱动的持续调优

```
90 天结果数据
    │
    ├──→ 权重调优：哪个维度对留存预测力最强？调高它的权重
    ├──→ 题库迭代：哪道题区分度下降了？替换为新题 A/B test
    ├──→ 知识图谱丰富：新的成功/失败案例并入图谱
    └──→ 协同过滤更新：发现问卷无法捕捉的隐式偏好模式
```

### 协同过滤解决什么问题

内容匹配（L2-L4）只能用"你填了什么"来匹配。但有些模式是问卷捕捉不到的，比如：
- "在问卷上表现为独立驱动的候选人，实际上在协作文化中留存更高"
- "同校毕业、经历相似的人，对同类公司的满意度强相关"

协同过滤通过分析**历史匹配结果**发现这些隐式模式，作为内容匹配的补充。
