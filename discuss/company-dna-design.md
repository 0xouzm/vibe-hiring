## 一、设计原则

### 1.1 与 Career DNA 的对称关系

匹配公式 `score_d = 1 - |C_d - J_d|` 需要两端输入：
- **C_d**（候选人侧）：来自 Career DNA 30 题标准题库，已完成
- **J_d**（企业侧）：来自 Company DNA，**本文档定义**

两套问卷共享同一套 8 维光谱模型，确保评分输出直接兼容 L2 匹配算法。

### 1.2 场景视角转换

Career DNA 问的是"**你**会怎么做"——测量个体行为偏好。
Company DNA 问的是"**你们团队通常**怎么做"——测量组织真实行为模式。

关键区别：
- 候选人回答反映**个人意愿**
- 企业回答反映**团队实际**（不是 HR 期望的样子）

### 1.3 防美化设计

企业问卷最大的风险是"文化美化"——HR 描述的是"理想文化"而非"真实文化"。
应对策略：多人填写 + 匿名机制 + HR-员工偏差检测。详见第六、七节。

### 1.4 题量控制

| 指标 | Career DNA | Company DNA | 理由 |
|------|-----------|-------------|------|
| 题数 | 30 | 20 | 企业侧需要 HR + 5名员工都填写，必须降低负担 |
| 填写时间 | ~25 分钟 | ~10 分钟 | 员工是被邀请参与，动机更弱，时间必须短 |
| 权重占比 | 60% | — | Company DNA 不单独计权，而是作为 J_d 输入 L2 公式 |

## 二、维度映射表

8 维光谱在企业视角下的重新定义：

| 维度 | Career DNA（个人视角） | Company DNA（团队视角） | 光谱两端 |
|------|----------------------|------------------------|---------|
| **Pace** | 你偏好快速迭代还是深度打磨？ | 团队通常快速上线还是打磨后发布？ | 快速迭代 ←→ 深度打磨 |
| **Collab** | 你偏好独立还是协作？ | 团队通常独立并行还是紧密协作？ | 独立驱动 ←→ 团队协同 |
| **Decision** | 你依赖数据还是直觉？ | 团队做决策时主要看数据还是经验？ | 数据驱动 ←→ 直觉判断 |
| **Expression** | 你表达直接还是含蓄？ | 团队文化鼓励直接表达还是含蓄委婉？ | 直接开放 ←→ 策略含蓄 |
| **Unc** | 你拥抱模糊还是追求明确？ | 团队通常在明确方向后行动还是边做边定？ | 拥抱模糊 ←→ 追求明确 |
| **Growth** | 你偏好专精还是广泛？ | 团队更看重深度专家还是跨领域通才？ | 专精深耕 ←→ 广泛涉猎 |
| **Motiv** | 你被使命还是回报驱动？ | 员工主要因什么留在团队？ | 使命价值 ←→ 回报激励 |
| **Execution** | 你偏好严格计划还是灵活应变？ | 团队通常如何跟踪和交付工作？ | 计划严谨 ←→ 灵活应变 |

## 五、填写流程

### 5.1 角色与流程

```
HR 创建企业档案
    │
    ├─→ HR 自己填写 Company DNA（20 题，~10 分钟）
    │
    ├─→ 邀请 5+ 名员工匿名填写（邮件/链接）
    │       │
    │       ├─→ 员工 1 填写（匿名）
    │       ├─→ 员工 2 填写（匿名）
    │       ├─→ ...
    │       └─→ 员工 N 填写（匿名）
    │
    └─→ 系统聚合所有回答 → 生成 Company DNA 评分
```

### 5.2 最低要求

| 要求 | 值 | 理由 |
|------|---|------|
| HR 填写 | 必须 | 提供管理视角的基线 |
| 员工最少人数 | 5 人 | 低于 5 人统计意义不足 |
| 员工角色要求 | 同团队/同部门 | 确保填写的是同一个文化单元 |
| 匿名性 | 强制匿名 | 消除员工的社会期望偏差 |
| 填写截止时间 | 邀请后 7 天 | 超时自动提醒，14 天未完成则标记为"验证不足" |

### 5.3 刷新机制

- 每 6 个月提醒重新填写（文化会变）
- 团队成员变动 > 30% 时自动触发重新填写
- 企业可主动发起重新填写

## 六、聚合算法

### 6.1 标准选择题聚合

对于每道标准选择题，每个选项映射到维度上的位置值（0-100）。
聚合 N 个填写者的回答时，使用**去极值加权中位数**：

```
输入：N 个填写者的维度位置值 {v₁, v₂, ..., vN}
      每个填写者的权重 {w₁, w₂, ..., wN}

Step 1: 去极值
  如果 N ≥ 7：去掉最高和最低各 1 个值
  如果 N < 7：不去极值

Step 2: 加权排序
  将剩余值按大小排序，累计权重

Step 3: 加权中位数
  找到累计权重达到 50% 的位置 → 该位置的值即为维度评分

权重设定：
  HR 权重 = 0.5（HR 有美化动机，降低权重）
  员工权重 = 1.0（匿名填写，更真实）
```

**为什么用加权中位数而非加权平均？**
- 中位数对极端值更鲁棒——一个极端乐观或极端悲观的员工不会拉偏整体结果
- 加权确保 HR 的声音不被完全忽略，但也不占主导

### 6.2 预算分配题聚合

预算分配题的每个选项直接产出百分比值。

```
Step 1: 收集每个填写者对每个选项的百分比
Step 2: HR 权重 0.5，员工权重 1.0
Step 3: 对每个选项取加权中位数
Step 4: 归一化（确保总和 = 100%）
```

### 6.3 强制排序题聚合

```
Step 1: 每个填写者的排序转换为 Borda 计分
  5 个选项：排第 1 得 5 分，排第 2 得 4 分，...，排第 5 得 1 分

Step 2: 加权求和
  每个选项的总分 = Σ(权重ᵢ × Borda分ᵢ)

Step 3: 按总分排序 → 得到团队的聚合排序
Step 4: 将排序映射到对应维度评分
```

## 七、真实度验证

### 7.1 Culture Authenticity Score（文化真实度评分）

每家企业会获得一个 0-100 的真实度评分，影响候选人看到的信息展示。

```
CAS = 0.40 × InternalConsistency
    + 0.35 × HREmployeeAlignment
    + 0.25 × ThirdPartyAlignment

其中：
  InternalConsistency   = 员工之间答案的一致程度
  HREmployeeAlignment   = HR 回答与员工聚合结果的偏差程度（偏差越大越低）
  ThirdPartyAlignment   = 企业评分与第三方数据（Glassdoor 等）的一致程度
```

### 7.2 Internal Consistency（内部一致性）—— 权重 40%

```
对于每个维度 d：
  std_d = 员工回答在该维度上的标准差

InternalConsistency = 1 - mean(std_d / max_range) for all d

解读：
  · 员工之间答案越一致 → 越高（说明文化明确）
  · 员工之间答案分散 → 越低（说明文化模糊或部门差异大）
```

> 注意：一致性低不一定是"造假"，也可能是团队确实文化多元。
> 系统会标注"Culture Clarity: High/Medium/Low"而非简单判断好坏。

### 7.3 HR-Employee Alignment（HR-员工偏差检测）—— 权重 35%

```
对于每个维度 d：
  hr_d    = HR 在该维度的回答
  emp_d   = 员工聚合后的维度评分
  gap_d   = |hr_d - emp_d|

HREmployeeAlignment = 1 - mean(gap_d / max_range) for all d

预警阈值：
  · gap_d > 25（任意维度）→ 标记该维度为"HR-员工认知分歧"
  · mean(gap_d) > 20 → 整体标记为"文化自我认知偏差"
```

**对候选人可见的信息：**
- 高偏差维度会在匹配报告中标注："Note: Team members' views on [dimension] vary from the official description. Worth exploring in interviews."
- 不会暴露具体数值，但提醒候选人面试时重点了解。

### 7.4 Third-Party Alignment —— 权重 25%

```
数据来源：Glassdoor、Blind、LinkedIn 评价等

对比逻辑：
  · 将第三方评价通过 NLP 提取维度倾向
  · 与 Company DNA 聚合结果对比
  · 偏差大 → 降低该维度的置信度

冷启动降级：
  · 如果该企业无第三方数据 → 该项权重分配给 InternalConsistency
  · 实际公式变为：CAS = 0.55 × IC + 0.45 × HREA
```

### 7.5 CAS 等级与展示

| CAS 范围 | 等级 | 候选人可见标签 | 含义 |
|----------|------|-------------|------|
| 85-100 | Gold | "Verified Culture" | 高一致性，HR 与员工高度一致 |
| 65-84 | Silver | "Culture Profile Active" | 基本可信，个别维度有偏差 |
| 40-64 | Bronze | "Culture Profile Pending" | 偏差较大，建议面试深入了解 |
| 0-39 | — | 不展示 Company DNA | 数据不可信，仅展示硬性条件 |

## 八、与 Career DNA 的评分兼容性

### 8.1 评分输出格式

Company DNA 经过聚合后，输出与 Career DNA 完全相同格式的 8 维评分：

```
{
  "pace": 72,        // 0-100，偏高 = 快速迭代
  "collab": 45,      // 偏低 = 偏独立
  "decision": 68,    // 偏高 = 偏数据驱动
  "expression": 81,  // 偏高 = 偏直接开放
  "unc": 55,         // 中间 = 平衡
  "growth": 38,      // 偏低 = 偏专精
  "motiv": 62,       // 偏高 = 偏使命驱动
  "execution": 74    // 偏高 = 偏计划严谨
}
```

### 8.2 输入 L2 匹配算法

```
score_d = 1 - |C_d - J_d| / scale_d

其中：
  C_d = 候选人 Career DNA 维度评分（来自 question-bank-full.md 30 题）
  J_d = 企业 Company DNA 维度评分（来自本文档 20 题 × N 人聚合）
```

每个维度额外附带置信度系数：
- Career DNA 侧：来自交叉验证一致性检测
- Company DNA 侧：来自 CAS 中的 InternalConsistency × HREmployeeAlignment

两侧置信度的乘积用于加权该维度在最终匹配分中的贡献。

## 九、未来扩展

1. **岗位级 DNA 叠加**：同一公司不同团队/岗位可以有不同的 Company DNA（团队级填写）
2. **AI 语音补充**：让员工做 5 分钟语音回答，NLP 提取更丰富的文化信号
3. **持续校准**：入职候选人 90 天后回答"这家公司实际如何"，持续校准 Company DNA
4. **Glassdoor 数据融合**：自动关联第三方评价，为 ThirdPartyAlignment 提供数据源
